## 한 줄 정의

그라운드 트루스(ground truth)는 모델의 예측과 비교할 기준이 되는, 사람이 확인했거나 실제로 관측된 "정답" 데이터를 말한다.

## 자세히

머신러닝 모델은 입력에 대한 답을 스스로 만들어내지만, 그 답이 맞았는지 틀렸는지 판단하려면 비교할 기준이 필요하다. 이 기준이 그라운드 트루스다. 이미지 분류라면 각 이미지에 사람이 직접 붙인 정답 라벨("고양이", "강아지")이, 스팸 탐지라면 실제로 스팸인지 아닌지 확인된 이메일 태그가 그라운드 트루스가 된다. 훈련 단계에서는 모델이 이 정답을 보고 학습하고(지도학습), 평가 단계에서는 모델의 예측을 그라운드 트루스와 대조해 정확도·재현율 같은 지표를 계산한다. [[혼돈행렬]]의 네 칸(TP/TN/FP/FN)도 결국 "예측이 그라운드 트루스와 일치했는가"를 기준으로 나눈 것이고, [[ROC]] 곡선의 TPR·FPR 역시 그라운드 트루스 대비 예측의 정확도를 임계값별로 추적한 값이다.

그라운드 트루스는 대개 사람의 수작업(라벨링)으로 만들어지기 때문에 비용과 품질이 큰 이슈다. AWS SageMaker에는 이 과정을 지원하는 **SageMaker Ground Truth**라는 관리형 라벨링 서비스가 있는데, 사람이 직접 라벨을 붙이는 작업을 워크플로로 관리해주고, 일부는 이미 라벨링된 데이터로 훈련한 모델이 자동으로 라벨을 붙이는 "능동 학습(active learning)"으로 비용을 줄여준다. 서비스 이름 자체가 "정답 데이터를 만드는 도구"라는 뜻을 그대로 담고 있다.

최근에는 RAG(검색 증강 생성)나 LLM 평가에서도 같은 개념이 쓰인다. RAGAS 같은 평가 프레임워크에서 그라운드 트루스는 "사람이 검수한 정답 답변"을 가리키며, 모델이 생성한 답변이나 검색된 컨텍스트가 이 정답과 얼마나 의미적으로 가깝고 관련 있는지를 컨텍스트 정밀도·컨텍스트 재현율·답변 의미 유사성 같은 지표로 측정하는 기준점이 된다.

주의할 점은 그라운드 트루스 자체도 완벽하지 않을 수 있다는 것이다(라벨러의 실수, 주관적 판단 차이 등). 그래서 "정답"이라기보다 "현재 확보한 가장 신뢰할 수 있는 기준"으로 이해하는 편이 정확하다.

## 예시

지도학습 데이터셋에서:

```python
X = images            # 입력
y_true = labels        # 그라운드 트루스 (정답 라벨)
y_pred = model.predict(X)  # 모델 예측

accuracy = (y_pred == y_true).mean()  # 그라운드 트루스와 비교해 정확도 계산
```
