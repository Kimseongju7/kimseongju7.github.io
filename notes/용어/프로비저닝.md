## 한 줄 정의
필요한 IT 자원(서버·스토리지·네트워크 등)을 요청에 맞춰 준비하고 사용할 수 있게 할당하는 과정.

## 자세히
프로비저닝(provisioning)은 "쓸 수 있게 미리 채비해 둔다"는 뜻이다. 서버 한 대를 예로 들면, 물리 장비나 가상 머신을 띄우고 OS·네트워크·권한·소프트웨어까지 설정해서 실제로 서비스가 돌아갈 수 있는 상태로 만드는 일련의 작업이 전부 프로비저닝에 해당한다. 반대로 더 이상 필요 없는 자원을 회수하는 것을 **디프로비저닝(deprovisioning) / 프로비저닝 해제**라고 한다.

클라우드에서 이 개념이 특히 중요한 이유는 자원을 **동적으로** 늘렸다 줄일 수 있기 때문이다. [[온프레미스]](직접 서버를 사서 운영하는 환경)에서는 트래픽이 몰릴 때를 대비해 장비를 미리 넉넉히 사둬야 하고, 한가한 시간대에는 그만큼 낭비가 생긴다. 반면 AWS 같은 클라우드에서는 요청이 많으면 그 순간 자원을 프로비저닝하고, 부하가 끝나면 바로 해제한 뒤 **쓴 만큼만 비용을 내는 종량제**로 운영할 수 있다. 커피숍에 비유하면, 점심시간에 점원을 즉시 늘렸다가 붐비는 시간이 끝나면 곧바로 내보내는 것과 같다.

프로비저닝은 손으로 콘솔에서 클릭해가며 할 수도 있지만, 규모가 커지면 **코드로 자동화**하는 것이 일반적이다. Terraform·CloudFormation 같은 IaC(Infrastructure as Code) 도구로 "어떤 자원이 몇 개 필요하다"를 코드로 선언하면, 도구가 그 상태에 맞게 자원을 생성·변경·삭제해 준다. 이렇게 하면 사람이 매번 수동으로 세팅할 때 생기는 실수와 편차를 줄이고, 같은 환경을 반복해서 재현할 수 있다.

## 예시
- **자동 확장([[Amazon-EC2-Auto-Scaling|Auto Scaling]])**: 웹 트래픽이 급증하면 EC2 인스턴스를 자동으로 여러 대 프로비저닝하고, 트래픽이 줄면 다시 해제해 비용을 절약한다.
- **IaC 선언 예시** (Terraform):

  ```hcl
  resource "aws_instance" "web" {
    ami           = "ami-0abcd1234"
    instance_type = "t3.micro"
  }
  ```

  `terraform apply` 를 실행하면 이 선언대로 EC2 인스턴스 한 대가 프로비저닝된다.
