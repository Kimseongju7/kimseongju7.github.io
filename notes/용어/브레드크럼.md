## 한 줄 정의
브레드크럼(breadcrumb)은 "홈 > 학습 > 용어" 처럼 현재 페이지가 사이트 계층 구조 안에서 어디에 있는지 보여 주는 경로 표시 내비게이션이다.

## 자세히
이름은 동화 〈헨젤과 그레텔〉에서 빵 부스러기(bread crumb)를 떨어뜨려 돌아갈 길을 표시한 데서 왔다. 웹에서는 사용자가 "지금 내가 사이트의 어느 깊이에 있고, 어떤 상위 경로를 거쳐 여기까지 왔는지"를 한눈에 알려 주는 수평 링크 목록을 가리킨다. 보통 페이지 상단, 제목 위쪽에 `홈 > 카테고리 > 현재 글` 같은 형태로 놓인다.

핵심 효과는 두 가지다. 첫째, **방향 감각**을 준다 — 깊은 페이지에 검색이나 외부 링크로 바로 들어온 방문자도 자기 위치와 상위 분류를 파악하고, 한 단계 위로 쉽게 올라갈 수 있다. 둘째, **클릭 수를 줄인다** — 브라우저 뒤로가기 대신 경로 중간의 상위 항목을 바로 눌러 이동할 수 있다. 보통 맨 끝(현재 페이지)은 링크가 아닌 일반 텍스트로 두고, 그 앞 단계들만 링크로 만든다.

브레드크럼은 대개 사이트의 **카테고리/폴더 계층**을 그대로 반영한다. 그래서 글에 카테고리를 잘 정해 두면 별도 작업 없이도 경로가 자연스럽게 만들어진다. 검색엔진([[SEO]])에도 도움이 되는데, 구글은 이 계층 정보를 읽어 검색 결과에 사이트 구조 형태(빵부스러기 경로)로 노출하기도 한다.

이 블로그가 쓰는 **[[Jekyll·Chirpy|Chirpy]] 테마**도 글 상단에 브레드크럼을 자동으로 보여 준다. 글 front matter의 `categories` 값을 그대로 경로로 풀어내기 때문에, 예를 들어 `categories: [학습, 용어]` 로 적은 글은 `Home > 학습 > 용어` 형태의 브레드크럼이 붙는다. Chirpy에서 카테고리를 2단계로 맞추라고 권하는 이유 중 하나도 이 경로가 너무 깊어지지 않게 하기 위함이다.

## 예시
front matter의 카테고리가 브레드크럼으로 이어지는 모습:

```yaml
---
title: 브레드크럼
categories: [학습, 용어]
---
```

→ 페이지 상단에 다음과 같이 표시된다:

```
Home  >  학습  >  용어
```

여기서 `Home` 과 `학습`, `용어` 는 클릭 가능한 링크(해당 분류 목록으로 이동)이고, 맨 끝의 현재 글 제목은 보통 링크 없는 텍스트로 남는다.
