## 한 줄 정의
하나의 물리 하드웨어 위에서 여러 개의 가상 머신(VM)을 만들어 나눠 쓰게 해 주고, 그 자원 배분을 관리하는 소프트웨어(또는 펌웨어) 계층.

## 자세히
하이퍼바이저(hypervisor)는 CPU·메모리·스토리지 같은 물리 자원을 추상화해서, 각 가상 머신에게 "나는 혼자 물리 컴퓨터 한 대를 쓰고 있다"고 착각하게 만든다. VMM(Virtual Machine Monitor)이라고도 부른다. 여러 VM이 같은 하드웨어를 공유하지만([[멀티테넌시]]), 하이퍼바이저가 각 VM을 서로 격리(isolation)해 주기 때문에 한 VM에서 일어난 일이 다른 VM에 영향을 주지 않는다. 자원을 어느 VM에 얼마나 줄지 스케줄링하는 것도 하이퍼바이저의 역할이다.

하이퍼바이저는 크게 두 종류로 나뉜다. **Type 1(베어메탈, native)** 은 하드웨어 위에 하이퍼바이저가 곧바로 올라가는 방식으로, 호스트 OS 없이 직접 하드웨어를 제어한다. VMware ESXi, Microsoft Hyper-V, Xen, KVM(리눅스 커널에 내장) 등이 여기에 해당하며, 성능과 안정성이 좋아 데이터 센터·클라우드 서버에서 주로 쓴다. **Type 2(호스티드)** 는 이미 깔려 있는 일반 OS(윈도우·macOS 등) 위에 응용 프로그램처럼 얹혀 돌아가는 방식으로, VirtualBox, VMware Workstation 같은 것이 대표적이다. 설치가 쉬워 개인 PC에서 테스트용으로 많이 쓰지만, 호스트 OS를 한 번 거치므로 성능 오버헤드가 있다.

클라우드가 "컴퓨터 한 대를 빌려 준다"는 개념을 가능하게 하는 핵심 기술이 바로 하이퍼바이저다. AWS EC2를 예로 들면, 물리 서버 한 대 위에서 하이퍼바이저가 여러 고객의 인스턴스를 동시에 돌리고 서로 격리해 준다. 덕분에 물리 장비를 잘게 쪼개 여러 사용자에게 "쓴 만큼만" 빌려 줄 수 있다. 참고로 요즘은 하이퍼바이저 전체가 아니라 OS 커널을 공유하며 더 가볍게 격리하는 **컨테이너**(Docker 등)도 널리 쓰이는데, VM보다 가볍지만 격리 수준은 하이퍼바이저 기반 VM이 더 강하다는 차이가 있다.

## 예시
- **AWS EC2**: 같은 물리 하드웨어 위에 여러 고객의 EC2 인스턴스가 동시에 구동되고(멀티테넌시), 이를 하이퍼바이저가 격리·관리한다. (AWS는 자체 개발한 Nitro 하이퍼바이저를 사용한다.)
- **개인 PC 가상화**: 맥북에서 VirtualBox(Type 2)를 깔고 그 위에 Ubuntu VM을 띄워 리눅스를 실험해 보는 상황.
