## 한 줄 정의

오류 임계값(error threshold)은 SageMaker Data Wrangler에서 데이터를 훈련/검증/테스트로 분할할 때, 지정한 분할 비율(예: 80/20)을 얼마나 정확하게 맞출지를 조절하는 값(0~1)이다.

## 자세히

Data Wrangler의 분할(Split) 기능은 훈련/검증/테스트 세 갈래로 데이터를 나누는데, 이때 각 분할의 비율을 퍼센트로 지정한다(세 비율의 합은 항상 1, 즉 100%가 되어야 함). 문제는 Data Wrangler가 대용량 데이터를 여러 파티션에 나눠 분산 처리하기 때문에, "정확히 80.000%"처럼 완벽한 비율을 맞추려면 전체 데이터를 다시 훑거나 셔플하는 등 추가 연산이 필요하다는 점이다. 오류 임계값은 이 "정확도 vs 처리 시간"의 절충점을 사용자가 직접 정하게 해주는 옵션이다.

값을 **0으로 설정**하면 지정한 비율을 오차 없이 정확히 맞춘 분할을 얻지만, 그만큼 처리 시간이 길어질 수 있다. 반대로 임계값을 **크게** 주면 처리는 빨라지지만 실제 분할 비율이 목표치에서 다소 벗어날 수 있다. 예를 들어 10,000행짜리 데이터를 80/20으로 나누면서 오류 임계값을 0.001로 지정하면, 정확히 8,000/2,000이 아니라 8,010/1,990이나 7,990/2,010처럼 목표 비율에 아주 가까운(하지만 완전히 같지는 않은) 결과가 나올 수 있다.

함께 나오는 **랜덤 시드(random seed)** 옵션은 분할의 재현성을 위한 것이다. 같은 시드를 쓰면 같은 데이터에 대해 항상 같은 분할 결과를 얻을 수 있어, 나중에 같은 실험을 다시 돌리거나 다른 사람과 결과를 맞춰볼 때 유용하다. 시드를 지정하지 않으면 Data Wrangler가 기본 랜덤 시드를 사용하며, 재현성이 특별히 중요하지 않은 실습이라면 비워 둬도 무방하다.

## 예시

- 오류 임계값 `0`: 80/20 분할 지정 시 정확히 8,000 : 2,000으로 나뉨 (대신 처리 시간 ↑)
- 오류 임계값 `0.001` (10,000행 기준): 8,010:1,990 또는 7,990:2,010처럼 목표 비율에 근접한 결과 (처리 시간 ↓)
