## 한 줄 정의
컴포넌트(Component)는 화면을 이루는 UI를 **독립적이고 재사용 가능한 조각**으로 나눈 단위로, 각 조각이 자신의 구조·모양·동작을 스스로 갖고 있어 레고 블록처럼 조립해 하나의 화면을 만든다.

## 자세히
전통적인 웹에서는 HTML, CSS, JavaScript를 파일 종류별로 따로 관리했다. 하지만 화면이 복잡해질수록 "이 버튼과 관련된 코드가 어디 흩어져 있지?"를 찾기 어려워진다. 컴포넌트는 발상을 뒤집는다. **파일 종류가 아니라 화면 조각(기능) 단위로** 코드를 묶는다. 예를 들어 '좋아요 버튼'이라는 컴포넌트 하나 안에 그 버튼의 모양(마크업), 스타일, 눌렀을 때의 동작이 함께 들어 있다. 그래서 그 버튼을 고치거나 다른 곳에 다시 쓰고 싶을 때 그 컴포넌트만 보면 된다.

컴포넌트의 핵심 가치는 **재사용성과 조합**이다. 헤더, 검색창, 카드, 버튼 같은 작은 컴포넌트를 만들어 두면, 이들을 겹겹이 쌓아 더 큰 컴포넌트(예: 상품 목록 페이지)를 만들 수 있다. 마치 함수를 조합해 프로그램을 만들듯, 컴포넌트를 조합해 화면 전체를 만든다. 같은 카드 디자인이 100개 필요해도 카드 컴포넌트 하나를 만들어 데이터만 바꿔 100번 재사용하면 된다. 이 "작게 나누고 조립한다"는 사고방식이 오늘날 [[React Doctor|React]]·Vue·Svelte 같은 프론트엔드 도구들이 공통으로 채택한 방식이다.

컴포넌트가 서로 데이터를 주고받는 방법도 정형화돼 있다. 부모 컴포넌트가 자식에게 값을 내려줄 때는 **props(속성)**를 쓰고, 컴포넌트가 스스로 기억하고 바꾸는 데이터는 **state(상태)**라 부른다. 예컨대 좋아요 버튼은 "눌렸는지 여부"를 state로 갖고, 눌릴 때마다 그 state가 바뀌면 컴포넌트가 **다시 그려진다(re-render)**. 이때 실제 [[DOM]]을 직접 건드리는 대신 [[Virtual DOM|가상 DOM]]으로 바뀐 부분만 골라 반영해 성능을 아끼는 것이 React식 접근이다. 즉 컴포넌트는 단순한 "화면 조각"을 넘어, 자기 상태를 갖고 그에 따라 스스로 화면을 갱신하는 **작은 자율 단위**다.

정리하면 컴포넌트는 (1) UI를 나누는 단위이자, (2) 재사용·조합의 블록이며, (3) props로 입력받고 state로 상태를 관리하는 작은 프로그램이다. [[SPA]]가 화면 전환 없이 앱처럼 동작하는 구조라면, 그 화면을 실제로 조립하는 재료가 바로 컴포넌트다.

## 예시
React에서 '좋아요 버튼'을 컴포넌트로 만든 예:

```jsx
// props로 상품 이름을 받고, state로 좋아요 여부를 스스로 관리한다
function LikeButton({ label }) {
  const [liked, setLiked] = React.useState(false);

  return (
    <button onClick={() => setLiked(!liked)}>
      {label}: {liked ? '❤️ 좋아요' : '🤍 좋아요'}
    </button>
  );
}

// 같은 컴포넌트를 데이터만 바꿔 여러 번 재사용
function App() {
  return (
    <div>
      <LikeButton label="사과" />
      <LikeButton label="바나나" />
    </div>
  );
}
```

`LikeButton` 하나를 만들어 두면 사과·바나나 어디든 재사용할 수 있고, 각 버튼은 자기 `liked` 상태를 따로 기억한다. 버튼을 누르면 그 컴포넌트만 다시 그려져 화면이 갱신된다.
