## 한 줄 정의
검색엔진이 내 페이지를 잘 이해하고 상위에 노출하도록 사이트를 다듬는 작업. (Search Engine Optimization, 검색엔진 최적화)

## 자세히
사람들은 구글·네이버 같은 검색엔진에 뭔가를 검색하고, 결과 목록의 위쪽 몇 개만 클릭한다. SEO는 "검색 결과에서 더 위에, 더 잘 보이게" 만들어 자연 유입(광고비 없이 들어오는 방문자)을 늘리는 활동이다. 검색엔진은 크롤러(봇)로 웹을 돌아다니며 페이지를 수집(크롤링)하고, 색인(인덱싱)한 뒤, 검색어와의 관련성·신뢰도를 따져 순위를 매긴다. SEO는 이 세 단계 각각이 잘 돌아가게 돕는 일이다.

크게 세 갈래로 나눈다. **온페이지(on-page)** 는 페이지 안에서 다루는 것으로, 제목 태그(`<title>`), 메타 설명, 헤딩 구조(H1~H3), 본문 키워드, 이미지 `alt` 텍스트, 읽기 좋은 URL(→ [[슬러그]]) 등이 해당된다. **테크니컬(technical) SEO** 는 검색엔진이 사이트를 잘 크롤링·색인하도록 하는 기술적 토대로, 사이트맵(`sitemap.xml`), `robots.txt`, 페이지 로딩 속도, 모바일 대응, HTTPS, 구조화 데이터(schema.org), 정규 URL(`canonical`) 등이 있다. **오프페이지(off-page)** 는 사이트 밖에서 쌓이는 신뢰로, 다른 좋은 사이트가 내 글을 링크해 주는 백링크가 대표적이다.

블로그 관점에서 실질적으로 중요한 건 대개 온페이지와 테크니컬이다. 좋은 제목과 설명을 붙이고, 글마다 명확한 주제와 키워드를 두고, 검색엔진이 읽기 좋은 URL과 사이트맵을 제공하는 것만으로도 기본은 갖춰진다. 이 블로그가 쓰는 Jekyll·Chirpy(→ [[Jekyll·Chirpy]])는 각 글의 front matter(`title`, `description`, `categories`, `tags`)로 메타 태그를 자동 생성하고 `sitemap.xml`도 만들어 주므로, front matter를 성실히 채우는 것이 곧 SEO다.

주의할 점은 SEO가 "검색엔진을 속이는 기술"이 아니라는 것. 키워드를 억지로 도배하거나(keyword stuffing) 내용 없는 페이지를 양산하면 오히려 순위가 떨어진다. 결국 핵심은 **사람에게 유용한 콘텐츠를 검색엔진이 이해하기 쉬운 형태로 제공**하는 것이다.

## 예시
Chirpy 블로그 글의 front matter — 이것만 잘 채워도 온페이지 SEO의 기본이 된다.

```yaml
---
title: SEO 기초 정리          # <title>, 검색 결과 제목
description: 검색엔진 최적화의 핵심 개념  # 메타 설명 = 검색 결과 요약문
categories: [학습, 용어]      # URL·빵부스러기 경로
tags: [seo, 블로그]          # 주제 키워드
---
```

검색엔진에게 "이 사이트의 모든 페이지 목록"을 알려주는 사이트맵:

```
https://kimseongju7.github.io/sitemap.xml
```

## 관련
- [[슬러그]] — 온페이지 SEO에서 다루는 읽기 좋은 URL 형식
- [[브레드크럼]] — 검색 결과 노출에도 영향을 주는 사이트 경로 UI 요소
- [[Jekyll·Chirpy]] — front matter만으로 메타 태그·sitemap.xml을 자동 생성해주는 이 블로그의 정적 사이트 엔진
- [[SSR]] — 검색엔진이 읽기 좋은 완성된 HTML을 내려줘 SEO에 유리한 렌더링 방식
- [[CSR]] — 크롤러가 빈 HTML만 보게 되어 SEO에 불리한 렌더링 방식
- [[MPA]] — 페이지마다 완성된 HTML을 내려줘 전통적으로 SEO에 강한 구조
- [[SPA]] — 크롤러가 빈 HTML만 볼 수 있어 SEO에 불리한 앱 구조
- [[메타프레임워크]] — SSR·SSG를 골라 써 SPA/CSR의 SEO 약점을 메워 주는 도구
