**[산학협력 프로젝트] 연구노트 글감 리스트업 및 8회차 초안 작성**
- ✨ 8월 18일 제출용 연구노트를 쓰기 위해, 볼트의 `notes/` 전체(용어·SWM AI Infra 교육·web 클립·일지)를 병렬 조사해 글감을 뽑았다. 작년(2025년) 연구노트 PDF에서 서식을 먼저 파악하려 했으나 텍스트 추출 도구가 없어 poppler를 설치해 읽었고, **개요 → 기술 섹션 2~4개 → 기록자/점검자 날짜(2주 간격, 점검은 기록일 +3일)** 구성에 A4 2단 조판 기준 페이지당 900~1,100자라는 기준을 확보했다.
- ✨ 기간을 5~8월로 잡고 마지막 회차가 8.18 이전에 오도록 05.02 → 08.08까지 2주 간격 8회차를 배치했다. 주제는 AI/ML 인프라 학습 축(클라우드 비용 모델 → ML 데이터 준비 → 모델 평가·튜닝 → 배포 전략 → MLOps·드리프트 → LLM 추론 파라미터 → RAG 청킹·벡터 인덱싱 → AI 에이전트 프로덕션 전환)으로, 과제명과 정합되게 정했다.
- ✨ 초안은 `2026_소프트웨어중심대학 산학협력 프로젝트/8월 18일 제출/연구노트_초안_김성주.md`에 8페이지 전부 작성했다. 프로젝트명·팀원명·특정 서비스 고유명은 전부 배제했고, "무엇을 배웠다"에서 끝나지 않도록 각 절 끝에 선택 기준과 판단 근거(예: 스팟 vs 약정형 선택 근거, 롤백 속도가 우선이면 [[블루그린배포|블루그린]])를 붙였다. 개념 정리 위주라 직접 겪은 트러블슈팅 기록으로 몇 편 교체할지는 보류하고 일단 이대로 두기로 했다.

**[개인] 공기업 후보 비교 양식 작성**
- ✨ 지원할 공기업을 추리기 위한 [[공기업 후보 비교|비교 양식 노트]]를 만들었다. 처음 잡았던 기준(사택·위치·잡플래닛 평점·연봉)이 전부 "들어간 뒤 삶의 질"에 몰려 있어서, **채용 가능성**(전산직 직렬 존재 여부, 연간 채용 규모, 필기 유형, 지역인재 목표제)과 **하는 일** 축을 추가했다. 조건이 아무리 좋아도 전산 직렬이 없으면 후보에서 빠지므로 이쪽이 사실상 1순위라고 봤다.
- ✨ 양식은 한눈에 보기 표 → 기관별 상세 블록 → 정리 세 덩어리로 구성하고, 전산직 칸을 **정규직 / 체험형 인턴**으로 분리했다. 인턴은 결국 정규직 전환 가점이 있느냐가 전부라 모집 시기와 가점 여부를 따로 적도록 했다. 이후 구내식당(유무·식대 지원·한 끼 가격)과 「하는 일」 항목을 추가했는데, 기관 소개문은 어디나 좋게 쓰여 있어서 *전산직이 실제로 자체 개발을 하는지 SI 발주·관리만 하는지*를 별도 줄로 분리했다.

## 배운 것

<!-- claude-recall-learn:rlatjdwn:start -->
**연구노트의 서식과 서술 밀도**
- ✨ 산학협력 프로젝트 연구노트는 페이지마다 개요로 문을 열고 기술 섹션 2~4개로 본문을 채운 뒤 기록자·점검자 날짜로 닫는 형태이고, 기록은 2주 간격·점검은 기록일 +3일로 찍힌다. 분량은 A4 2단 조판 한 페이지에 900~1,100자 정도라, 새로 쓸 때 이 숫자를 기준으로 잡으면 조판 후 넘치거나 비는 일을 줄일 수 있다. 양식 파일(hwp)만 보고 짐작하는 것보다 작년 실제 제출본을 읽어 톤과 밀도를 맞추는 편이 훨씬 정확했다.

**폴더 이름만 보고 단정하지 말 것**
- ✨ `5월 8일 제출` 폴더 이름만 보고 "이미 연구노트를 낸 적이 있으니 주제가 겹칠 수 있다"고 판단했는데, 실제로 열어보니 개인정보 동의서·청렴서약서·재학증명서 같은 행정 서류뿐이고 연구노트는 없었다. 이름은 내용의 근거가 아니다 — 확인 한 번이면 끝날 일을 추측으로 채우면, 있지도 않은 제약을 만들어 계획을 왜곡하게 된다.

**공공기관 유형 분류**
- ✨ 「공공기관의 운영에 관한 법률」상 분류는 갈림길이 두 개다. 먼저 규모 요건(정원 300명·수입 200억·자산 30억 이상)을 넘으면 공기업/준정부기관, 못 넘으면 기타공공기관. 그다음 자체수입비율이 50% 이상이면 공기업, 미만이면 준정부기관이다. 공기업은 다시 시장형(자산 2조 이상 + 자체수입 85% 이상, 한전·인천공항공사)과 준시장형(LH·코레일·도로공사)으로, 준정부기관은 기금관리형(국민연금공단·신용보증기금)과 위탁집행형(건강보험공단·코트라)으로 나뉜다. 취준 입장에서 실제로 갈리는 건 성과급 변동성으로, 공기업·준정부기관은 기재부 경영평가(S~E) 등급에 성과급이 연동된다.

**공공기관 연봉·직급 수치를 읽는 법**
- ✨ 알리오의 **평균연봉은 근속이 긴 기관일수록 높게 나와 왜곡**되므로, 신입 지원자는 초임을 따로 봐야 한다. 그리고 고졸 채용 초임이 함께 실려 있어 대졸 수치와 헷갈리기 쉽다(IBK의 경우 일반정규직 약 5,777만원 vs 고졸 약 4,350만원).
- ✨ 「관리 6급/7급」 같은 표기는 **공무원 급수가 아니라 기관 자체 사내 직급**이고, 신입이 몇 급으로 들어가는지가 기관마다 제각각이다(HUG는 대졸이 관리6급·고졸이 관리7급, 다른 곳은 5급이 대졸 신입). 따라서 급수를 기관끼리 비교하는 건 의미가 없다. 또 모집분야에 직렬을 명시하는 기관(IBK `금융일반·디지털·IT`, 심평원 `전산직` 등 30곳가량)과 아예 안 적는 기관이 갈리므로, 후보는 직렬을 명시한 곳부터 보는 편이 효율적이다.
