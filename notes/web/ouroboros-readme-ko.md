## 한국어판의 프레이밍

한국어 README는 "프롬프트를 멈추세요. 명세를 시작하세요."라는 슬로건 아래 Ouroboros를 **명세 우선 AI 개발 시스템**으로 소개한다. 인상적인 한 줄: *"AI는 무엇이든 만들 수 있다. 어려운 건 무엇을 만들어야 하는지 아는 것이다."*

- 대부분의 AI 코딩은 **출력**이 아니라 **입력** 단계에서 실패한다. 병목의 원인은 AI의 능력이 아니라, 뭘 만들지 덜 정한 채 시작하기 때문이다. "Ouroboros는 기계가 아닌 인간을 바로잡습니다."
- 소크라테스식 질문법과 온톨로지 분석으로, 단 한 줄의 코드를 쓰기 전에 사용자의 숨겨진 가정을 드러낸다.
- 철학 도식: *Wonder → "어떻게 살아야 하는가?" → "'삶'이란 무엇인가?" → 온톨로지* (소크라테스). "인증 버그를 고치자"는 "이게 근본 원인일까, 아니면 증상일까?"가 된다. **온톨로지 질문이야말로 가장 실용적인 질문이다.**

Double Diamond 설명도 한국어판 표현이 간명하다: 첫 다이아몬드는 소크라테스적(질문을 넓히고, 온톨로지가 또렷해질 때까지 좁힘), 두 번째는 실용적(설계 옵션을 넓히고, 검증된 결과물로 좁힘). "이해하지 못한 것은 설계할 수 없기 때문입니다."

## 영문판과 다른 세부 수치 (스냅샷 차이)

한국어판은 영문판과 다른 시점의 수치를 담고 있다.

- 내부 구조 헤더가 **"18개 패키지 · 166개 모듈 · 95개 테스트 파일 · Python 3.14+"** 로 표기 (영문판은 Python >= 3.12 요구로 표기)
- Brownfield: **"12개 이상의 언어 생태계에서 15종의 설정 파일 스캔"** 이라고 구체적으로 명시 (영문판은 "여러 언어 생태계 자동 감지"로 일반화)
- 지원 런타임을 Claude Code, Codex CLI, Kiro CLI 중심으로 서술 (영문판은 OpenCode, Hermes, Gemini, Copilot, Pi까지 확장)
- 명령어 표에 `ooo setup`이 "MCP 서버 등록 (1회)"로 설명되고, `ooo status`는 "드리프트 감지 + 세션 추적"으로 표기

## 한국어판 고유 섹션: 실시간 모니터링 (TUI)

영문 README에는 없는 **터미널 대시보드** 섹션이 있다. `ooo run`이나 `ooo evolve`를 돌릴 때 별도 터미널에서 같이 띄워 실시간 워크플로를 볼 수 있다.

```
# 설치 및 실행
uvx --from 'ouroboros-ai[tui]' ouroboros tui monitor

# 로컬 설치된 경우
uv run ouroboros tui monitor
```

| 키 | 화면 | 표시 내용 |
| --- | --- | --- |
| `1` | **Dashboard** | 단계 진행률, 수용 기준 트리, 실시간 상태 |
| `2` | **Execution** | 타임라인, 단계별 출력, 상세 이벤트 |
| `3` | **Logs** | 레벨별 색상 구분, 필터링 가능한 로그 뷰어 |
| `4` | **Debug** | 상태 인스펙터, 원시 이벤트, 설정 |

자세한 내용은 저장소의 TUI 사용 가이드(`docs/guides/tui-usage.md`)를 참고.

## 기억해둘 한국어판 문장들

- "뱀이 한 바퀴를 돌고 나면 다음 바퀴는 다릅니다. 전보다 더 많이 알게 되니까요."
- "우로보로스(자기 꼬리를 삼키는 뱀)는 그냥 상징이 아닙니다. 우로보로스는 아키텍처 그 자체입니다."
- Interview는 느낌으로 끝내지 않고 **수학적 계산** 점수가 기준 밑으로 내려와야 끝난다 (Ambiguity ≤ 0.2).
- "시작이 곧 끝이고, 끝이 곧 시작이다." / "뱀은 반복하지 않는다 — 진화한다."

아홉 에이전트의 핵심 질문 번역도 재치 있다: Researcher는 *"근거 있어?"*, Simplifier는 *"돌아가는 것 중 제일 단순한 건?"*, Architect는 *"처음부터 다시 짜면 정말 이렇게 갈까?"*.
