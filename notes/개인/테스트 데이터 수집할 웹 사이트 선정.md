"수정 연혁이 남는" 문서를 테스트 데이터로 수집하기 위해 후보 사이트를 정리한다. 크게 **법령**과 **논문(프리프린트)** 두 갈래이며, 각 사이트가 API·대량 다운로드를 얼마나 열어 두었는지가 선정 기준이다.

## 법령 — 국가법령정보센터

- **URL:** <https://law.go.kr/>
- 법령을 검색한 뒤 **신구법 비교**를 누르면 무엇이 바뀌었는지 확인할 수 있고, 다운로드도 가능하다.
- 수정 연혁을 확인해서 가져올 수는 있을 것으로 보인다.
- 다만 사이트 구조가 복잡한 편이라, 수집 시 그에 맞는 적절한 수집 전략을 세워야 한다.

- **활용 방향:** 여기서는 "수정본을 얼마나 뽑아낼 수 있는지"보다, 수집한 수정본을 "어떻게 정리해서 보여줄지"를 테스트할 때 써볼 수 있을 듯하다.

## 논문의 수정 연혁을 볼 수 있는 곳

논문은 크게 세 가지 유형의 플랫폼에서 수정 이력을 추적할 수 있다.

- **프리프린트(Preprint) 서버** — 정식 출판 전의 논문을 연구자들이 미리 공유하는 곳. 논문이 수정될 때마다 버전을 업데이트해 올리기 때문에(예: v1, v2, v3) 논문의 발전 과정을 볼 수 있다. 분야별 대표 서버로는 [arXiv](https://arxiv.org/)(물리학·수학·컴퓨터과학), [bioRxiv](https://www.biorxiv.org/)(생물학), [SSRN](https://www.ssrn.com/)(사회과학·경제학) 등이 있다.
- **오픈 피어 리뷰(Open Peer Review) 저널** — '오픈 사이언스' 기조에 따라 초기 투고본, 심사자(Reviewer) 코멘트, 저자의 수정 내역을 모두 투명하게 공개하는 학술지. F1000Research, eLife, BMJ Open 등이 대표적이다.
- **오픈 사이언스 플랫폼** — 연구 설계 단계부터 데이터 수집, 논문 작성까지의 전 과정을 기록·공유하는 플랫폼. 대표적으로 **OSF(Open Science Framework)** 는 프로젝트 단위로 연구 히스토리가 남아 작업 과정을 추적하기 좋다.

## 데이터 수집 관점에서 본 각 서버

### 1. arXiv — 가장 개방적이고 강력한 API 지원

컴퓨터과학·물리학·수학 분야의 최대 프리프린트 서버로, 데이터 수집에 가장 훌륭한 환경을 제공한다.

- **메타데이터 API:** 논문 제목·초록·저자·제출일 등의 메타데이터를 검색·추출할 수 있는 공식 REST API와 OAI-PMH를 제공한다.
- **Python 라이브러리:** `arxiv`라는 공식/비공식 패키지가 활성화되어 있어, `pip install arxiv` 만으로 메타데이터를 긁어오고 PDF를 다운로드할 수 있다.
- **대용량 일괄 다운로드(Bulk Data Access):** 수만 건 이상을 통째로 분석(텍스트 마이닝 등)하고 싶다면, AWS S3 버킷을 통해 전체 PDF와 소스 파일(LaTeX)을 대량으로 내려받는 공식 경로도 열려 있다.

### 2. bioRxiv & medRxiv — 메타데이터 API 지원

생명과학·의학 분야의 프리프린트 서버로, 공식 메타데이터 API를 제공한다.

- **API 형태:** 날짜 구간·DOI·서버(bioRxiv/medRxiv)를 지정해 JSON 또는 XML 형식으로 메타데이터를 100건씩 페이징해 받아올 수 있다. (예: `https://api.medrxiv.org/details/biorxiv/YYYY-MM-DD/YYYY-MM-DD/0/json`)
- **PDF 다운로드:** API 자체는 '메타데이터' 전용이다. 따라서 PDF를 대량으로 받으려면 API로 논문의 URL이나 DOI를 먼저 수집한 뒤, 그 링크로 별도의 스크래핑(다운로드) 코드를 작성해 내려받아야 한다. (R 언어의 경우 `medrxivr` 같은 패키지가 이 과정을 돕는다.)

### 3. SSRN — 공식 API 없음, 스크래핑 엄격히 차단

사회과학·경제학 중심의 SSRN은 현재 대형 학술 출판사 엘스비어(Elsevier) 소유이며, **프로그램을 이용한 자동화된 접근과 대량 다운로드를 매우 엄격하게 차단**하고 있다.

- 공식 다운로드 API나 메타데이터 API가 없다.
- 웹 크롤링(BeautifulSoup, Selenium 등)을 시도해도 캡차(CAPTCHA)나 봇 탐지 시스템에 의해 IP가 쉽게 차단된다. 따라서 SSRN에서 논문 히스토리나 데이터를 대량 수집하는 것은 기술적으로 매우 까다롭고 규정 위반의 소지가 있다.

### 4. OSF Preprints (Open Science Framework)

오픈 사이언스 플랫폼인 OSF 역시 다양한 학문 분야의 프리프린트를 호스팅하며, 매우 훌륭한 **OSF REST API**를 제공한다. 메타데이터 검색은 물론 논문 파일(PDF 등)의 다운로드 링크까지 API로 안정적으로 확보할 수 있다.

## 정리

- IT/컴퓨터나 생물학 쪽 논문을 분석한다면 **arXiv나 bioRxiv의 API**를 활용해 논문의 연혁과 PDF 원문을 원활하게 수집할 수 있다.
- 반면 **사회과학 분야(SSRN)** 를 타깃으로 한다면 자동화된 수집이 어렵다는 점을 염두에 두어야 한다.
