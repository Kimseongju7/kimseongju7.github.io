## 배운 것

<!-- claude-recall-learn:castleown:start -->
- 🤖 [Jekyll](/posts/jekyll-chirpy/)은 정적 사이트 생성기 — 빌드 시점에 모든 페이지를 HTML로 미리 만들어두고 방문자는 완성된 HTML을 그대로 받음 (빠르고 무료, 보안 취약점 적음)
- 🤖 GitHub Pages 발행은 push → 빌드 → 배포 + Fastly CDN 캐시를 거쳐 보통 1~3분, 길면 5~10분 걸림. Actions 탭에서 빌드 성공(초록 체크) 여부를 먼저 확인
- 🤖 한글 헤더의 내부 앵커 링크는 [kramdown](/posts/kramdown/)이 만드는 실제 헤더 ID와 달라 htmlproofer가 깨진 링크로 잡음 → 앵커 대신 평문화가 안전
<!-- claude-recall-learn:castleown:end -->

<!-- claude-recall-learn:rlatjdwn:start -->
- 🤖 Claude Code의 **skill**은 자동 호출(description 매칭으로 Claude가 스스로 판단) + 사용자 직접 호출(`/skill-name`) 둘 다 가능. command는 사용자 직접 호출 전용
- 🤖 같은 "커스텀 프롬프트" 개념을 도구마다 다르게 부름 — Claude Code는 slash command(`.claude/commands/*.md`), Codex는 skill(`~/.codex/skills/<name>/SKILL.md`)
- 🤖 ECC(everything-claude-code)는 단일 플러그인이 아니라 skills 182개 + agents 48개 등을 담은 엔지니어링 워크플로우 통합 번들
- 🤖 `~/.claude`를 git으로 버전 관리(dotfiles 패턴)하면 여러 컴퓨터의 Claude 설정을 동기화할 수 있음 — 단 토큰·비밀 같은 민감 정보는 올리지 않도록 분리 필요
<!-- claude-recall-learn:rlatjdwn:end -->
