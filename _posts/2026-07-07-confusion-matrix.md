## 한 줄 정의

혼돈행렬(confusion matrix)은 분류 모델이 실제 정답과 예측을 얼마나 맞고 틀렸는지, 그 조합별 개수를 표로 정리한 것이다.

## 자세히

이진 분류를 예로 들면, 모델의 예측과 실제 정답을 각각 긍정(양성)/부정(음성)으로 나눠 네 칸짜리 표를 만든다.

- **TP (True Positive)**: 실제 긍정을 긍정이라고 옳게 판단
- **TN (True Negative)**: 실제 부정을 부정이라고 옳게 판단
- **FP (False Positive)**: 실제 부정인데 긍정이라고 잘못 판단 (1종 오류)
- **FN (False Negative)**: 실제 긍정인데 부정이라고 잘못 판단 (2종 오류)

정확도(accuracy)만 보면 놓치는 게 많다. 예를 들어 사기 거래가 1%뿐인 데이터에서 무조건 "정상"이라고만 찍어도 정확도는 99%가 나오지만, 정작 사기(긍정)는 하나도 못 잡아낸 모델이다. 혼돈행렬은 이런 클래스 불균형 상황에서 모델이 정확히 어떤 종류의 실수를 하고 있는지(놓치는 쪽인지, 오탐하는 쪽인지) 드러내 준다.

혼돈행렬의 네 칸으로부터 재현율(recall, TP/(TP+FN)), 정밀도(precision, TP/(TP+FP)), F1 점수 같은 세부 평가지표가 계산된다. 이 지표들은 [ROC](/posts/roc/) 곡선이 TPR·FPR을 임계값별로 훑는 것과 달리, 하나의 임계값(보통 0.5)에서 고정된 스냅샷을 보여준다.

## 예시

암 진단 모델을 100명에게 적용한 결과:

|          | 예측: 양성 | 예측: 음성 |
|----------|-----------|-----------|
| 실제: 양성 | TP = 8    | FN = 2    |
| 실제: 음성 | FP = 5    | TN = 85   |

- 정확도 = (8+85)/100 = 93%
- 재현율 = 8/(8+2) = 80% (실제 환자 10명 중 8명을 찾아냄)
- 정밀도 = 8/(8+5) = 62% (양성 판정 13명 중 8명만 진짜 환자)

scikit-learn에서는 한 줄로 구한다:

```python
from sklearn.metrics import confusion_matrix

confusion_matrix(y_true, y_pred)
# array([[85,  5],
#        [ 2,  8]])
```
