## 한 줄 정의
DOM(Document Object Model)은 HTML 문서를 브라우저가 **객체 트리 구조**로 표현한 것으로, JavaScript가 이 트리를 통해 문서의 내용·구조·스타일을 읽고 바꿀 수 있게 해주는 인터페이스다.

## 자세히
브라우저가 HTML 파일을 받으면 그 텍스트를 그대로 화면에 그리는 게 아니라, 태그 하나하나를 **노드(node)** 객체로 바꿔 부모-자식 관계를 가진 트리로 만든다. 예를 들어 `<body>` 안에 `<div>`, 그 안에 `<p>` 가 있으면 body 노드 아래 div 노드, 그 아래 p 노드가 매달리는 식이다. 이 트리 전체가 DOM이고, 최상단에는 `document` 객체가 있다.

DOM이 중요한 이유는 **정적인 HTML과 동적인 화면 사이의 다리** 역할을 하기 때문이다. HTML 자체는 한 번 그려지면 끝인 문서지만, `document.querySelector(...)` 로 원하는 노드를 찾고 `.textContent` 를 바꾸거나 `.appendChild()` 로 새 노드를 붙이면 화면이 즉시 갱신된다. 우리가 보는 "버튼을 누르면 목록이 늘어나는" 같은 상호작용은 전부 JavaScript가 DOM을 조작한 결과다.

주의할 점은 DOM 조작이 **비싸다**는 것이다. 노드를 바꿀 때마다 브라우저는 레이아웃을 다시 계산(reflow)하고 다시 그리는(repaint) 과정을 거친다. 그래서 React 같은 라이브러리는 실제 DOM을 직접 건드리는 대신 메모리상의 가벼운 복제본인 **[가상 DOM(Virtual DOM)](/posts/virtual-dom/)** 을 두고, 바뀐 부분만 골라 실제 DOM에 반영해 성능을 아낀다. 또한 DOM은 HTML 소스 코드 그 자체가 아니라 **브라우저가 파싱한 결과의 살아있는 표현**이라는 점도 헷갈리기 쉽다 — 개발자 도구의 Elements 탭에서 보는 것이 바로 현재 DOM 상태이며, JavaScript로 바꾸면 원본 HTML은 그대로여도 DOM은 달라진다.

## 예시
```html
<ul id="list">
  <li>사과</li>
</ul>
```

```javascript
// id가 list인 노드를 DOM 트리에서 찾는다
const list = document.querySelector('#list');

// 새 <li> 노드를 만들어 트리에 붙인다 → 화면이 즉시 갱신됨
const item = document.createElement('li');
item.textContent = '바나나';
list.appendChild(item);
```
위 코드가 실행되면 HTML 파일은 그대로지만, DOM에는 `<li>바나나</li>` 가 추가되어 화면에 "사과, 바나나"가 보인다.
