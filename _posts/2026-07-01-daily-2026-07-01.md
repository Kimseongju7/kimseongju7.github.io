## 배운 것
#### react-doctor
- [react-doctor](/posts/react-doctor/)는 AI 코딩 에이전트(Claude Code, Cursor 등)나 개발자가 만들어내는 잘못된 React 패턴을 결정론적(deterministic)으로 잡아내는 코드 스캐너다.
- LLM이 확률적으로 코드를 짠다면, react-doctor는 그 결과물을 규칙 기반으로 검사해 안티패턴을 차단한다. 나쁜 코드 패턴 체크리스트를 갖고 있어서, 마치 사람이 코드리뷰 하듯 내 코드를 훑으며 해당되는 부분을 하나하나 대조한다.
- 사용법은 3가지: ① 한 번 검사하기 ② AI에게 미리 가르치기 ③ CI에 붙여 PR 올릴 때마다 댓글 달게 하기.
#### Loop Engineering
- 한 줄로 정의하면: AI에게 직접 명령하지 말고, AI에게 자동으로 명령을 내리는 시스템(루프)을 설계하라.
- 지금까지는 사람이 AI에게 일을 시켰다면, 이제는 사람이 루프 밖으로 빠지고 시스템이 AI를 굴린다. 사람이 '엔진' 역할에서 '엔진 설계자' 역할로 옮겨가는 것이다.
1. Discovery (발견) — 이번에 뭘 할지 스스로 찾는다 (예: 실패한 CI, 열린 이슈, 최근 커밋 읽기). 일을 받는 게 아니라 찾게 하는 게 핵심. 루프 품질의 천장을 결정.
2. Handoff (위임) — 찾은 일을 작업 에이전트에 넘긴다. 작업마다 격리된 git worktree를 줘서 서로 안 부딪히게.
3. Verification (검증) — 다른 에이전트가 결과를 채점. "안 돼(no)"라고 말할 수 있는 단계. 가장 빼먹기 쉽고, 가장 빼면 안 되는 동작.
4. Persistence (저장) — 결과를 대화창 밖(파일·보드)에 남긴다. PR, 티켓, 상태 파일.
5. Scheduling (스케줄링) — 타이머로 반복시킨다. 이게 있어야 "한 번 실행"이 진짜 "루프"가 됨.

이걸 실현하는 6가지 부품(Parts): 자동화(Automations), 워크트리(Worktrees), 스킬(Skills), 커넥터(Connectors=MCP), 서브에이전트(Sub-agents), 메모리(Memory).
