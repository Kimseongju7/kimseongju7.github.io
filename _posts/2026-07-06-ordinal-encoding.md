## 한 줄 정의

서수 인코딩(ordinal encoding)은 범주 사이에 순서(등급)가 있는 변수를 그 순서를 보존하도록 0, 1, 2, … 같은 정수로 바꾸는 인코딩 방식이다.

## 자세히

컴퓨터는 문자열을 그대로 학습할 수 없기 때문에 범주형 변수는 반드시 숫자로 바꿔야 한다. 이때 범주가 "순서형(ordinal)"인지 "명목형(nominal)"인지가 어떤 인코딩을 써야 할지를 가른다. 순서형 변수는 학력(`고졸 < 대졸 < 대학원졸`), 등급(`하 < 중 < 상`), 만족도(`매우불만족 < … < 매우만족`)처럼 범주 사이에 크기·순위 관계가 실제로 존재하는 변수다. 서수 인코딩은 이 순서를 그대로 숫자의 크기 관계로 옮겨 담는다 — 예를 들어 `고졸=0, 대졸=1, 대학원졸=2`.

이 점에서 서수 인코딩은 **레이블 인코딩**과 자주 혼동된다. 둘 다 범주를 정수 하나로 매핑한다는 구현은 똑같지만, 레이블 인코딩은 그냥 등장 순서나 알파벳 순으로 임의로 번호를 매기는 것이라 숫자 크기에 의미가 없는 반면, 서수 인코딩은 처음부터 "이 순서가 실제 크기 관계를 반영한다"는 것을 전제로 순서를 지정해서 매핑한다는 점이 다르다. 반대로 성별·국적·혈액형처럼 범주 간 순위가 없는 **명목형** 변수에 서수 인코딩(또는 레이블 인코딩)을 잘못 쓰면, 모델이 "국적 코드 2가 국적 코드 1보다 크다"는 식으로 존재하지 않는 크기 관계를 학습해버리는 문제가 생긴다. 이런 명목형 변수에는 대신 원-핫 인코딩을 쓰는 것이 일반적이다.

서수 인코딩의 장점은 원-핫 인코딩과 달리 새 컬럼을 늘리지 않아 차원이 그대로 유지된다는 것이다. 범주 개수가 아주 많은 순서형 변수(예: 별점 1~10점)에서 특히 유용하다. 다만 순서 간 "간격"이 실제로 균등하다고 보장하지는 않는다 — 예컨대 만족도를 0~4로 인코딩해도 "매우만족"과 "만족"의 차이가 "불만족"과 "매우불만족"의 차이와 똑같은 크기라는 뜻은 아니며, 이 점은 선형 모델을 쓸 때 특히 주의해야 한다.

## 예시

```python
from sklearn.preprocessing import OrdinalEncoder

# 순서를 명시적으로 지정 — 지정하지 않으면 알파벳/등장 순으로 임의 매핑되므로 주의
encoder = OrdinalEncoder(categories=[["고졸", "대졸", "대학원졸"]])
df["학력_인코딩"] = encoder.fit_transform(df[["학력"]])
# 고졸 → 0, 대졸 → 1, 대학원졸 → 2
```

pandas로도 간단히 매핑할 수 있다.

```python
order = {"하": 0, "중": 1, "상": 2}
df["등급_인코딩"] = df["등급"].map(order)
```
