## 요약

폴더 상세 페이지의 버전·분기 노드 연결 구조를 설계하고, 노션 디자인 시스템을 받아 하이파이 시안을 제작했다.

## 한 일

- 폴더 상세 페이지 노드 연결리스트 설계에서 버전 노드 연결 · 분기 묶음 후보군이 중앙 상단 버전 줄에서 v2-1 같은 분기를 어디 끼울지(= v3? v2-2?)가 모호함 → 한 버전의 앞뒤 분기를 **한 박스로 묶어 하나의 노드**로 보고, 메인 노드끼리만 직선으로 잇는 방향으로 설계. 분기 노드들의 경우 클릭 시 에만 나타나는 식으로 토글로 설정. 호버도 계획해봤지만 모바일 환경 같은 경우에는 분기가 있다는 사실을 알기 힘들어 토글로 선택
- ![분기 노드 연결 구조 설계 스케치](/assets/img/posts/daily-2026-06-29/branch-node-design.png)
- https://github.com/VoltAgent/awesome-design-md/blob/main/design-md/notion/DESIGN.md 에서 노션 디자인시스템을 찾은 후 `npx getdesign@latest add notion` 를 이용해 노션의 디자인 시스템을 다운 받음
- 클로드 디자인에 노션 디자인 시스템 등록하고 해당 디자인 시스템과 미리 만든 와이어프레임을 이용해 하이파이 시안 제작

## 배운 것

- 노션 디자인 시스템 분석 중 어두운 인디고 블루가 튀어나옴. 노션 사용 시에는 본 적이 없어 다시 보니 마케팅 페이지·랜딩 페이지에서 단 한 번만 사용하는 [히어로](/posts/hero/) 값이라고 함
- 위 깃허브에서 가져온 시스템은 노션 앱의 디자인 시스템이 아닌 노션 웹 페이지 디자인 시스템이라고 함. 다시 찾아봐야 함
<!-- claude-recall-learn:start -->
- 🤖 프롬프트 1·2번은 동일한 "끝까지 자율 완수" 오케스트레이션 지시문을 공유(병렬 에이전트 분배 → 추적 → 종합 → 검증 → 리뷰/커밋)하고 앞의 Task만 다름. 3번은 팁/참고자료, 4~8번은 "Act as ..." 역할 부여형 단발 프롬프트
- 🤖 `Notion_Design_System` / `Github` 패키지는 같은 AI 디자인 도구가 만든 자매 산출물(동일 구조) — Notion은 DESIGN.md 분석을 입력으로 "Geul-Hub" 가상 브랜드를 생성, Github는 실제 GitHub Primer 기반
