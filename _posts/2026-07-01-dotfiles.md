## 한 줄 정의
dotfiles(닷파일)는 이름이 점(`.`)으로 시작해 평소엔 숨겨져 있는, 개발 환경·도구의 개인 설정 파일들을 통칭하는 말이다.

## 자세히
유닉스 계열(macOS·리눅스)에서는 파일명이 `.` 으로 시작하면 `ls` 같은 기본 명령에서 보이지 않는 "숨김 파일"이 된다. `.bashrc`, `.zshrc`, `.gitconfig`, `.vimrc` 처럼 셸·에디터·도구가 자기 설정을 읽어오는 파일들이 대부분 이 형태라서, 이런 설정 파일들을 한데 묶어 **dotfiles** 라고 부른다. 보통 홈 디렉터리(`~`)에 흩어져 있다.

dotfiles가 따로 이름까지 붙어 화제가 되는 이유는, 이걸 **하나의 git 저장소로 모아 관리**하는 관행 때문이다. 새 컴퓨터를 받았을 때 셸 설정·단축키·alias·테마를 처음부터 다시 잡으려면 번거롭다. dotfiles를 저장소(흔히 GitHub의 `dotfiles` 리포)에 올려두고, 새 기기에서 클론한 뒤 [심볼릭 링크](/posts/symbolic-link/)(`ln -s`)나 설치 스크립트로 홈 디렉터리에 연결하면 "나만의 작업 환경"을 몇 분 만에 복원할 수 있다. 즉 환경 설정을 코드처럼 버전 관리하고 재현 가능하게 만드는 것이 핵심이다.

관리 방식은 여러 가지다. 직접 짠 셸 스크립트로 심링크를 거는 단순한 방법부터, [GNU Stow](https://www.gnu.org/software/stow/)·[chezmoi](https://www.chezmoi.io/)·[yadm](https://yadm.io/) 같은 전용 도구를 쓰는 방법까지 있다. 기기마다 다른 값(이메일, 토큰 등)은 템플릿이나 로컬 전용 파일로 분리해서, 공개 저장소에 비밀값이 올라가지 않도록 주의해야 한다.

## 예시
홈 디렉터리에서 숨김 파일까지 보려면 `-a` 옵션을 준다:

```bash
$ ls -a ~
.zshrc        .gitconfig     .vimrc
.ssh/         .config/       Documents/
```

흔한 dotfiles와 역할:

- `.zshrc` / `.bashrc` — 셸 시작 시 실행되는 설정 (alias, PATH, 프롬프트)
- `.gitconfig` — git 사용자 이름·이메일·alias·기본 동작
- `.vimrc` — Vim 에디터 설정
- `.ssh/config` — SSH 접속 호스트별 설정

git으로 관리하는 가장 단순한 형태:

```bash
# ~/dotfiles 저장소에 설정 파일을 모아두고
git clone https://github.com/me/dotfiles ~/dotfiles
# 홈 디렉터리에 심볼릭 링크로 연결
ln -s ~/dotfiles/.zshrc ~/.zshrc
ln -s ~/dotfiles/.gitconfig ~/.gitconfig
```
