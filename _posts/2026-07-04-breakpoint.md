## 한 줄 정의
브레이크포인트(breakpoint)는 반응형 웹에서 "이 화면 너비를 넘으면(못 미치면) 레이아웃을 다르게 보여 준다"고 정해 두는 기준 화면 폭이다.

## 자세히
같은 웹페이지라도 폭이 넓은 데스크톱과 좁은 스마트폰에서 똑같은 레이아웃을 쓰면 어느 한쪽은 불편해진다. 그래서 "화면이 이 정도로 좁아지면(넓어지면) 배치를 바꾸자"라고 선을 긋는데, 그 선이 바로 브레이크포인트다. 레이아웃이 한 형태에서 다른 형태로 "끊어져(break)" 바뀌는 지점이라는 뜻이다. 예를 들어 3단 그리드를 화면이 좁아지면 1단으로 접거나, 가로로 놓였던 메뉴를 햄버거 버튼으로 바꾸는 전환이 이 지점에서 일어난다.

기술적으로는 CSS의 **미디어 쿼리(media query)** 로 구현한다. `@media (min-width: 768px) { ... }` 처럼 "화면 너비가 768px 이상일 때만 이 스타일을 적용" 같은 조건을 걸고, 그 경계값(여기서는 768px)이 브레이크포인트가 된다. 흔히 쓰는 값은 모바일·태블릿·데스크톱을 가르는 대략 576 / 768 / 992 / 1200px 근처인데, 이는 부트스트랩 같은 프레임워크가 정해 둔 관행일 뿐 절대적인 숫자는 아니다. 요즘은 특정 기기 크기에 맞추기보다 **콘텐츠가 보기 나빠지는 지점**에서 브레이크포인트를 잡으라고 권한다.

작성 방향에는 두 갈래가 있다. **모바일 우선(mobile-first)** 은 좁은 화면 스타일을 기본으로 두고 `min-width` 로 넓어질 때 규칙을 더해 가는 방식이고, **데스크톱 우선** 은 반대로 넓은 화면을 기본으로 두고 `max-width` 로 좁아질 때 덜어 낸다. 오늘날은 모바일 트래픽이 많고 스타일이 단순한 것에서 복잡한 것으로 쌓이는 편이 관리하기 쉬워 모바일 우선이 표준처럼 쓰인다.

참고로 **브레이크포인트라는 말은 두 곳에서 쓰인다.** 위의 반응형 웹 의미가 하나이고, 다른 하나는 **디버깅**에서 "프로그램 실행을 이 줄에서 잠깐 멈춰라"라고 코드에 찍어 두는 정지점(breakpoint)이다. 둘 다 "여기서 무언가가 바뀐다/멈춘다"는 뉘앙스는 같지만 맥락이 전혀 다르니, 화면 폭 이야기인지 디버거 이야기인지로 구분하면 된다. 이 노트는 반응형 웹 쪽을 다룬다.

## 예시
모바일 우선으로 카드 그리드를 짜고, 768px 이상에서 3단으로 펼치는 예:

```css
/* 기본(모바일): 1단 세로 나열 */
.cards {
  display: grid;
  grid-template-columns: 1fr;
  gap: 16px;
}

/* 브레이크포인트: 화면이 768px 이상이면 3단으로 */
@media (min-width: 768px) {
  .cards {
    grid-template-columns: repeat(3, 1fr);
  }
}
```

→ 스마트폰에서는 카드가 위아래로 한 줄씩 쌓이고, 화면 폭이 768px을 넘어서는 순간 3개가 가로로 나란히 배치된다.
