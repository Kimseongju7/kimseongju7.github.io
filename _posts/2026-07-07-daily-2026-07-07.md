## 배운 것

<!-- claude-recall-learn:rlatjdwn:start -->
- **Gateguard 훅(everything-claude-code)의 동작 방식**
  - 세션당 첫 Bash 호출과 파일별 첫 Edit/Write를 "Fact-Forcing Gate"로 한 번씩 차단해, 실행 전에 현재 요청과 명령의 목적을 명시하도록 강제한다. 옵시디언은 `claude -p "/스킬명"`으로 스킬마다 새 세션을 띄우기 때문에 매 실행마다 이 게이트가 새로 발동하며, `ECC_GATEGUARD=off` 환경변수로 특정 실행 경로만 선택적으로 끌 수 있다.
- **Tailwind CSS v4의 CSS-first 설정**
  - v4부터는 `tailwind.config.js` 없이 CSS 안에서 `@theme { ... }` 블록으로 토큰을 정의하는 게 공식 표준이다(v3의 JS 기반 설정을 대체하며, `@config`로 하위 호환은 유지). 이 프로젝트는 `DESIGN.md`(스펙) → `tokens.css`(`@theme inline`으로 CSS 변수 노출) → `bg-primary` 같은 시맨틱 유틸리티 순으로 파이프라인을 구성해, 컴포넌트에 raw 색상값(`bg-blue-500`)이 못 들어가게 강제하고 있다.
