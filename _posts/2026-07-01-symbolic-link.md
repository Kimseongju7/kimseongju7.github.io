## 한 줄 정의
심볼릭 링크(symbolic link, symlink)는 "다른 파일·디렉터리의 경로를 가리키는 바로가기" 역할을 하는 특수한 파일이다.

## 자세히
심볼릭 링크는 실제 데이터를 담는 대신, **다른 경로를 가리키는 포인터**만 들고 있는 파일이다. 윈도우의 "바로가기"와 비슷하게, 링크를 열거나 읽으면 운영체제가 알아서 가리키는 원본(target)으로 따라간다. 유닉스 계열에서는 `ln -s 원본 링크이름` 으로 만든다(`-s` 가 symbolic의 약자).

여기서 **하드 링크(hard link)** 와의 차이를 알아두면 좋다. 하드 링크는 같은 실제 데이터(inode)에 이름을 하나 더 붙이는 것이라 원본/사본 구분이 없고, 원본을 지워도 데이터가 살아 있다. 반면 심볼릭 링크는 **경로(이름)를 가리킬 뿐**이라, 원본을 옮기거나 지우면 링크는 아무 데도 가리키지 못하는 깨진 링크(dangling link)가 된다. 대신 디렉터리도 가리킬 수 있고, 다른 파일시스템·디스크를 넘어서도 만들 수 있다는 게 심볼릭 링크의 장점이다.

쓰임새는 다양하다. 여러 위치에서 같은 파일을 공유하고 싶을 때, 긴 경로에 짧은 별칭을 달고 싶을 때, 버전이 바뀌어도 같은 이름으로 접근하게 하고 싶을 때(예: `python` → `python3.12`) 자주 쓴다. 특히 [dotfiles](/posts/dotfiles/) 관리에서 핵심 도구인데, 설정 파일을 git 저장소 한곳에 모아두고 홈 디렉터리에는 심링크만 걸어두면, 원본은 버전 관리되면서도 도구들은 평소 경로(`~/.zshrc` 등)에서 설정을 읽을 수 있다.

## 예시
심볼릭 링크를 만들고 확인하는 흐름:

```bash
# 원본 파일을 가리키는 심링크 생성: ln -s <원본> <링크이름>
$ ln -s ~/dotfiles/.zshrc ~/.zshrc

# ls -l 로 보면 화살표(->)로 가리키는 대상이 보인다
$ ls -l ~/.zshrc
lrwxr-xr-x  1 user  staff  22 Jul  1 00:00 /Users/user/.zshrc -> /Users/user/dotfiles/.zshrc

# 원본을 지우면 깨진 링크가 된다 (가리킬 대상이 없음)
$ rm ~/dotfiles/.zshrc
$ cat ~/.zshrc
cat: /Users/user/.zshrc: No such file or directory
```

`l` 로 시작하는 권한 표시(`lrwxr-xr-x`)와 `->` 가 심볼릭 링크임을 알려준다.
