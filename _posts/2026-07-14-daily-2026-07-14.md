	## 요약

트위터에 북마크해 둔 아티클들을 옵시디언으로 옮겨 정리하고, Claude Code 도구 환경(Codex 플러그인·find-skills)을 점검했다. 맥 미니의 배경 화면만 깜빡이는 증상도 진단했다.

## 한 일

- 그동안 트위터에서 북마크해 둔 아티클들을 정리했다. 옵시디언으로 옮겨 요약을 읽었고, [프롬프트를 작성해주는 스킬](/posts/prompt-master-skill/)을 설정했다. 나중에 프로젝트를 하나 더 시작할 때는 [우로보로스](/posts/ouroboros-agent-os/)를 활용해보려 한다.
- 문서를 OCR로 마크다운으로 변환해주는 도구도 많이 보였다. 나중에 필요하면 검토해보려 한다.

<!-- claude-recall-done:castleown:start -->
**[castleown] Codex(GPT) 플러그인 설정 확인**
- ✨ Claude Code 안에서 [GPT 기반 Codex를 보조로 쓰는 플러그인](/posts/openai-codex-plugin-cc/) 설정 상태를 점검했다. Codex CLI 설치·ChatGPT 계정 로그인이 이미 끝나 있어 바로 사용 가능한 걸 확인했고, `/codex:rescue`(작업 위임)와 리뷰 게이트가 어떤 기능인지 파악했다. 리뷰 게이트(멈추기 전 매번 Codex 검수 강제)는 선택 사항이라 끈 상태로 두기로 했다.

**[castleown] find-skills 스킬 설치**
- ✨ skills.sh로 `find-skills` 스킬을 전역 설치했다. `~/.agents/skills/find-skills`에 깔려 Amp·Cline·Codex 등 60개 이상 에이전트에 심볼릭 링크로 연결됐다. Eve·PromptScript 2건은 전역 설치를 지원하지 않아 건너뛴 부분 실패였지만, Claude Code에선 정상 동작하는 걸 확인했다.

**[castleown] 맥 미니 배경 깜빡임 진단**
- ✨ 실행 중인 창은 멀쩡한데 뒤 바탕화면만 검게 깜빡이는 증상을 진단했다. 창은 정상이므로 케이블 신호 불량보다 화면을 그리는 시스템 레이어 문제로 좁혔고, USB-C 모니터 1대 직결 조건에 맞춰 케이블 교체 → 다른 Thunderbolt 포트 → 주사율 60Hz → HDR 끄기 → `killall Dock` 순서의 진단안을 정리했다.

## 배운 것

-

<!-- claude-recall-learn:castleown:start -->
**Codex 플러그인이 제공하는 것**
- ✨ Claude Code 안에서 GPT(Codex)를 보조 엔진으로 쓸 수 있다. `/codex:rescue`로 막힌 문제의 조사·수정·구현을 위임하고, 리뷰 게이트를 켜면 작업을 마치고 멈추기 전 매번 Codex가 결과물을 검수한다 — Claude가 쓴 코드를 GPT가 교차 검증하는 자동 이중 체크다.

**스킬 전역 설치와 세션 반영 시점**
- ✨ 전역(global) 스킬 설치는 대상 에이전트가 그 기능을 지원해야 가능하다 — Eve·PromptScript처럼 미지원이면 건너뛴다. 또 새로 설치한 스킬은 세션 시작 시점에 읽히기 때문에, 설치 직후 바로는 안 잡히고 세션을 재시작해야 활성화된다.

**바탕화면만 깜빡일 때 원인**
- ✨ 창은 정상인데 배경 레이어만 깜빡이면 케이블 신호 불량 같은 하드웨어보다 WindowServer/GPU·대역폭 쪽 문제일 가능성이 높다. 특히 USB-C 1대 직결은 저가·충전 전용 케이블의 대역폭 부족이 흔한 원인이라, 배경 갱신 순간 검은 프레임이 낀다 — 순정/USB4 인증 케이블, 60Hz, HDR 끄기가 유효한 대처다.
