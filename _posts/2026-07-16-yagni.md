## 한 줄 정의
"You Aren't Gonna Need It"(그거 필요 없을 거야) — 지금 당장 필요하지 않은 기능이나 추상화는 미리 만들지 말라는 소프트웨어 설계 원칙.

## 자세히
YAGNI는 익스트림 프로그래밍(XP)에서 나온 원칙으로, "나중에 필요할 것 같다"는 예측만으로 기능을 미리 구현하지 말고 실제로 필요해지는 시점에 구현하라는 뜻이다. "혹시 몰라서" 만들어둔 설정 옵션, 범용 인터페이스, 확장 포인트는 대부분 예측이 빗나가고, 쓰이지 않는 코드는 유지보수 비용과 이해 부담만 늘린다.

이 원칙이 중요한 이유는 미래를 예측하는 비용이 생각보다 크기 때문이다. 아직 요구사항이 확정되지 않은 상태에서 미리 설계한 추상화는 실제 요구사항과 어긋날 가능성이 높고, 그럴 경우 오히려 리팩터링을 더 어렵게 만든다. 반대로 필요할 때 그때그때 단순하게 구현하고, 반복되는 패턴이 실제로 나타나면 그 시점에 추상화하는 편이 훨씬 안전하다.

YAGNI는 종종 KISS(Keep It Simple)나 DRY(Don't Repeat Yourself)와 함께 언급되지만 강조점이 다르다. KISS가 "복잡하게 만들지 마라", DRY가 "중복을 없애라"라면, YAGNI는 "필요하지도 않은 걸 미리 만들지 마라"에 초점을 둔다. 예를 들어 세 줄이 비슷하다고 성급히 공통 함수로 뽑는 것도, 아직 반복이 확인되지 않은 상태에서 추상화를 미리 준비하는 것도 YAGNI 위반에 해당할 수 있다.

## 예시
- 아직 다국어 지원 요청이 없는데 모든 문자열을 i18n 라이브러리로 감싸두는 것 → YAGNI 위반
- API 응답 형식이 하나뿐인데 "나중에 XML도 지원할 수 있으니" 직렬화 전략 패턴을 미리 설계하는 것 → YAGNI 위반
- 지금 필요한 건 필드 하나 추가뿐인데, 앞으로 필드가 늘어날 걸 대비해 플러그인 시스템을 먼저 만드는 것 → YAGNI 위반
