## 배운 것

- ML 파이프라인 과정 전반에 대한 개괄적인 내용을 배웠다.
- SageMaker가 이 파이프라인의 인프라와 기능을 대부분 제공한다는 것을 알게 됐다.

<!-- claude-recall-learn:rlatjdwn:start -->
**Claude Code 권한 모드**
- ✨ "accept edits on"은 매번 허용받는 모드가 아니라 그 반대 — 파일 편집(Edit/Write)을 매번 물어보지 않고 자동 승인하는 모드. normal(기본)이 매번 확인, plan mode는 읽기 전용, bypass permissions는 거의 모든 것을 자동 승인. Shift+Tab으로 normal → accept edits → plan mode 순환.

**ROC 곡선 / AUC**
- ✨ 이진 분류 모델이 임계값을 바꿔가며 얼마나 잘 구별하는지를 TPR(재현율) vs FPR로 그린 그래프로, 왼쪽 위에 붙을수록 좋은 모델이고 대각선이면 랜덤 수준. 곡선 아래 면적을 AUC(0~1)로 요약해 비교하는 게 실무 관행이며, 클래스 불균형(사기 탐지 등)에서 accuracy보다 신뢰도 높음. SageMaker에서는 XGBoost/Linear Learner 훈련 로그의 `validation:auc`, Autopilot 리더보드, Model Monitor 품질 리포트에서 등장.

**서수 인코딩**
- ✨ 순서가 있는 범주형 변수(학력·등급 등)를 그 순서를 보존하며 0, 1, 2…로 바꾸는 방식. 레이블 인코딩과 구현은 비슷하지만 레이블 인코딩은 임의 순번인 반면 서수 인코딩은 실제 크기 관계를 반영한다는 차이가 있고, 순위 없는 명목형 변수에 잘못 쓰면 모델이 가짜 크기 관계를 학습하므로 그런 경우엔 원-핫 인코딩을 써야 함.

**오류 임계값 (SageMaker Data Wrangler)**
- ✨ 데이터 분할 비율(80/20 등)을 얼마나 정확히 맞출지 정하는 0~1 값. 0으로 두면 정확한 분할이 되지만 전체 데이터를 다시 훑어야 해 처리 시간이 늘고, 값을 키우면 처리는 빨라지지만 실제 비율이 목표치에서 살짝 벗어날 수 있음(예: 10,000행 80/20 분할에서 0.001이면 8,010:1,990 같은 근사값). 시간과 정확도 사이의 트레이드오프를 제어하는 옵션.
