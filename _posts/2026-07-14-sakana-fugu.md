**Sakana Fugu**는 강력하고 다양한 모델 풀(pool)을 동적으로 편성·조율해 높은 성능을 내는 멀티에이전트 시스템을, 단 하나의 OpenAI 호환 API로 제공하는 서비스다. 사람이 도메인 지식으로 팀 구성·역할·워크플로를 미리 정하는 대신, Fugu가 학습을 통해 사람이 떠올리기 어려운 고효율 협업 패턴으로 에이전트를 조합한다.

## 핵심 특징

- **하나의 API로 여러 모델을 최적 활용** — 전문 특화 모델 풀을 단일 API로 사용한다. 태스크별 모델 선택·전환은 Fugu가 담당하므로 API 관리 부담을 줄이면서 가성비를 높인다.
- **복잡한 태스크에서 우수한 성능** — 코딩·추론(리즈닝) 등 품질이 중요한 워크플로용으로 설계됐고, 전문 에이전트를 연계해 복잡한 태스크에 더 확실하고 신뢰할 수 있는 답을 낸다.
- **유연한 에이전트 선택** — 데이터·프라이버시·컴플라이언스·조직 요건에 맞춰 모델 풀에서 특정 프로바이더나 모델을 제외(옵트아웃)할 수 있다.

## 기반 기술: ICLR 2026 논문 2편

Fugu는 학습 기반 모델 오케스트레이션에 관한 ICLR 2026 논문 두 편, **TRINITY**와 **Conductor**를 기반으로 한다. 두 연구는 손으로 설계한 워크플로에 의존하지 않고, 태스크마다 전문 에이전트를 어떻게 편성·라우팅·조율할지 시스템 스스로 학습할 수 있음을 보여준다. 자세한 내용은 [테크니컬 리포트](https://arxiv.org/abs/2606.21228) 참고.

- **TRINITY: 진화형 LLM 코디네이터** ([논문](https://arxiv.org/abs/2512.04695)) — 경량 진화형 코디네이터가 여러 LLM을 여러 턴에 걸쳐 통솔한다. 각 모델에 Thinker(사고)·Worker(실행)·Verifier(검증) 역할을 부여하고, 코딩·수학·추론·지식 태스크에 맞춰 작업을 적응적으로 배분한다.
- **Conductor: 자연어 에이전트 오케스트레이션 학습** ([논문](https://arxiv.org/abs/2512.04388)) — 강화학습으로 훈련되어 자연어 기반 협조 전략을 스스로 발견한다. 에이전트 간 소통 패턴과 초점을 좁힌 프롬프트를 설계해, 다양한 LLM 풀이 난도 높은 추론 벤치마크에서 단일 모델을 능가하게 만든다.

## 두 가지 모델: Fugu와 Fugu Ultra

둘 다 하나의 OpenAI 호환 API로 제공되며, 통합 코드를 바꾸지 않고 전환할 수 있다.

- **Fugu** — 성능과 레이턴시의 균형. 일상 작업용 기본 모델로, Codex 같은 도구에 넣어 코딩·코드 리뷰에 쓰거나 응답성 좋은 챗봇을 구동할 수 있다. 풀에서 특정 에이전트를 제외해 데이터·프라이버시·컴플라이언스 제약을 맞출 수 있다.
- **Fugu Ultra** — 성능 최적화. 더 깊은 전문 에이전트 풀을 조율해 어렵고 중요한 문제에서 답변 품질을 극대화한다(그만큼 응답 시간은 길어짐). 얼리 유저들은 Kaggle 대회, 논문 재현, 사이버보안 분석, 문헌·특허 조사에 활용 중이다. Ultra는 성능을 위해 풀이 고정이며, 옵트아웃은 Fugu에서만 가능하다.

## 정성 평가 사례

프론티어 3종(Gemini 3.1 Pro high, Opus 4.8 max, GPT 5.5 xhigh)을 Model A/B/C로 익명화해(예시마다 매핑을 바꿈) 비교한 사례들:

- **AutoResearch · BPB (LLM 학습 레시피 개선)** — AutoResearch(Karpathy et al.) 프레임워크로 소형 GPT의 학습 레시피를 자율 개선. H100 GPU 1장에서 약 14시간 동안 123회 실험을 수행했고, 배치 크기·모델 깊이·학습률·옵티마이저 설정 등에서 스스로 개선점을 발견했다. Fugu-Ultra가 최고 평균 BPB 0.9774 ± 0.0019로 Model C(0.9781), Model B(0.9793), Model A(0.9822)를 앞섰고, 최고 단일 실행은 0.9748.
- **가나 소식(仮名消息) 읽기 순서 추정** — 산라시가키(散らし書き, 흩어쓰기)로 쓰인 1610년 호슌인(芳春院, 1547–1617)의 서한을 대상으로, 문자 바운딩 박스와 대략적 규칙만 주고 읽기 순서 추정 코드를 작성하게 한 실험. NED(전문가 정답과의 정규화 편집거리 기반, 1.0이 완전 일치) 기준 Fugu-Ultra 0.80 대 Model A 0.24, Model B도 비슷하게 낮았고 Model C는 유효한 코드를 내지 못했다. (서한 소장: 게이오대학 시도문고)
- **루빅스 큐브 솔버** — 순수 Python으로 라이브러리 없이 솔버를 작성하게 하고 랜덤 스크램블 300개로 평가. Fugu-Ultra와 Model A만 300개 전부 해결(Model B·C는 실행 시 크래시, 0/300). 평균 수(手)는 Fugu-Ultra 19.72 대 Model A 19.76으로 모두 최적 수준이며, Fugu-Ultra가 더 길었던 큐브는 하나도 없었다(7승 293무 0패).
- **CAD 메커니컬 아이리스** — 카메라 조리개처럼 여러 날개가 연동해 중앙 구멍을 여닫는 기계식 아이리스를 CAD로 생성. Fugu Ultra의 결과물만 날개가 바깥 핀을 축으로 회전하며 명확히 개폐됐고, 다른 모델들은 틈 발생, 약한 링크 기구, 불완전한 폐쇄 등의 문제를 보였다.
- **블라인드폴드 체스** — 보드 표시 없이 게임 전체를 기억하며 두는 4연전에서, Fugu가 프론티어 모델 3종과 2100 Elo Stockfish 엔진을 모두 체크메이트로 꺾었다.
- **주식 트레이딩(순차 의사결정)** — 익명화된 단일 종목 50주 데이터로, 미래를 보지 않고 주간 시장 데이터와 피드백만으로 매수·보유·매도를 결정. $10,000로 시작해 동일 파이프라인 5회 실행 결과 Fugu-Ultra는 $11,943.22 ± $633.86(평균 +19.43%)를 기록했고, 다른 프론티어 모델은 모두 +15% 미만. (일반적 트레이딩 성능 입증이 아닌 순차 의사결정 비교 목적이며, 과거 성과가 미래를 보장하지 않는다.)

## 요금제

**토큰 플랜(종량제)** — 고부하·프로덕션 워크로드용. 종량제 토큰은 월정액 플랜 토큰보다 높은 우선순위로 처리된다.
- Fugu: 에이전트 1개 활성 시 해당 기반 모델의 표준 요금만 부과. 여러 에이전트 활성 시에도 요금을 합산하지 않고, 관여한 최상위 모델 기준 단일 요금만 청구.
- Fugu Ultra(fugu-ultra-20260615, 100만 토큰당): 입력 $5, 출력 $30, 캐시 입력 $0.50. 컨텍스트 272K 초과 시 $10 / $45 / $1.00.

**구독 플랜(월정액)** — 캐주얼한 일상 사용용. 모든 티어에서 Fugu와 Fugu Ultra 둘 다 사용 가능.
- Standard $20/월 — 가벼운 일상 사용, 기본 사용량
- Pro $100/월 — 집중 작업 세션용, Standard의 10배
- Max $200/월 — 장시간 고부하 워크로드용, Standard의 20배
- 2026년 7월 말까지 가입 시 가입 티어의 2개월째 무료.

## FAQ 요점

- OpenAI 호환 API라 기존 클라이언트·코딩 하네스를 엔드포인트만 바꿔 쓰면 되고 SDK 마이그레이션이 필요 없다.
- 새 프론티어 모델이 공개되면 약 2주간 업데이트된 Fugu 모델을 훈련·평가한 뒤 순차 배포할 예정.
- 토큰 사용량과 비용은 요청 단위로 보고되어 실시간 지출 모니터링과 사전 비용 예측이 가능하다.
- 학습 데이터 사용은 콘솔에서 언제든 옵트아웃 가능.
- Fugu가 어떤 모델을 선택해 어떻게 조율하는지(라우팅 정보)는 독자 기술이라 비공개.
- 일본 외 지역에서도 사용 가능하나 EU/EEA 회원국에는 서비스를 제공하지 않으며, 그 외 지역도 네트워크 환경이나 현지 규제로 이용이 제한될 수 있다.
