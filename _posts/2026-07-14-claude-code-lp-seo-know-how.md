Claude Code에 일본어로 지시만 해서 [SEO](/posts/seo/)에 강한 기업 랜딩페이지(LP)를 약 2시간 만에 만들고, "Claude Code 법인 도입" 검색 1위 → 누적 문의 100건 초과(첫 3주 약 70건) → 첫 달 수주 약 3,200만 엔까지 이어진 과정을 공개한 글. 핵심은 예쁜 페이지가 아니라, 놔둬도 검색에서 사람이 오고 문의가 계속 생기는 "구조"를 한 장에 담는 것이었다.

## LP에 담은 구조

- **검색에서 발견되게 하기** — SEO, 구조화 데이터, 블로그 자동 갱신
- **온 사람에게 가치를 주고 연락처 받기** — 진단, 화이트페이퍼, 해설 영상(전부 AI 제작)
- **연락처가 들어오면 자동으로 움직이기** — 자동 회신 메일, 일정 조율 자동화, 담당자 Slack 알림
- **신뢰 쌓기** — 활용 사례, X·note 임베드

## "2시간"의 진실

- 작업 로그상 13:35 착수, 15:24에 LP 골격 완성. 첫날 커밋 21회.
- 다만 2시간 만에 완성된 것은 어디까지나 골격이고, 이 시점엔 검색해도 거의 나오지 않았다.
- 검색에 잡히게 하는 SEO 구현과 문의로 잇는 구조는 이후 며칠에 걸쳐 추가했다.
- "AI를 쓰면 2시간에 전부 완성"이 아니라, AI로 제작 시간을 대폭 압축하고 사람은 설계와 개선에 집중할 수 있게 됐다는 것이 실태다.

## 검색에 강해지기 위한 3가지 토대

1. **빠르고 순수한 HTML** — 표시를 빠르게 하고 Google이 읽기 쉬운 HTML로. 톱 이미지는 498KB에서 46KB까지 압축.
2. **누가 썼는지 기계에 전달** — 운영 회사, 저자, 서비스, 기사의 관계를 구조화 데이터로 연결해 검색엔진이 내용을 올바르게 이해하게 함.
3. **기사를 지속 추가하고 내부 링크로 연결** — LP 한 장으로 끝내지 않고 중심이 되는 완전 가이드와 관련 기사를 연결, 사이트 전체로 전문성을 전달하는 구조.

단, 검색 1위는 "Claude Code 법인 도입" 키워드에서의 자체 관측이며, 모든 검색에서 1위라는 뜻이 아니고 순위는 시기·환경에 따라 변동한다.

## 블로그가 매일 자동으로 쌓이는 구조

- 파이프라인: 소재 수집 → AI 집필 → 섬네일 생성 → 안전 체크 → 합격한 기사만 공개 → 검색엔진에 통지 → 일일 리포트
- 현재 매일 아침 본류 기사 2건 + 매일 저녁 뉴스 기사 1건, 하루 3건 페이스로 기사가 추가되고, 주간 단위로 기존 기사를 리라이트한다.
- 품질 관리는 AI의 주관적 점수에만 의존하지 않고, 문장 형식·최소 글자 수·본문 URL 실재 여부 같은 기계적으로 판정 가능한 안전 체크를 공개 전에 통과시킨다.
- AI에 쓰게 하는 것 이상으로 "사고 나는 기사를 공개하지 않는 구조"를 만드는 것이 중요했다.

## 진단과 화이트페이퍼로 방문을 문의로 전환

- LP에 응답 내용으로 자사의 AI 활용도를 5축으로 시각화하는 **AI 조직 진단**을 구현. 간이 결과는 그 자리에서 표시하고, 연락처를 입력한 사람에게는 상세 결과를 자동 발송. 이후 일정 조율, Zoom URL 발행, 담당자 알림까지 자동화.
- 또 하나의 입구는 **화이트페이퍼**. Claude Code 법인 도입과 보안을 정리한 자료를 원고→슬라이드→PDF→해설 영상까지 AI로 제작. 자료 요청이 오면 PDF, 해설 영상, 상담 링크를 자동 회신.
- 중요한 것은 자동화 범위의 명확화: 자동화한 것은 문의 접수, 자료 전달, 면담 예약까지. 면담·제안·클로징은 지금도 사람이 한다. AI에 입구를 맡기고 사람은 대화에 집중 — 이 선긋기가 소수 인원으로도 상담을 쌓을 수 있었던 이유 중 하나.

## 실제 성과

- LP에 활용 사례 32건 + X 게시물·note 임베드로 "제대로 발신하는 사람"이라는 신뢰를 축적.
- 문의 누적 100건 초과(첫 3주 약 70건)
- 유입원이 판명된 것 중 약 55%가 검색(SEO) — 광고비 제로, 가장 성장 중인 유입
- 수주 이미 약 3,200만 엔, 상담 전망(수주+진행 중)은 7,000~8,000만 엔 규모(수주율 58%)

## 진짜 중요했던 것은 LP가 아니라 "흐름"의 설계

기사가 늘어난다 → 검색에서 사람이 온다 → 진단·화이트페이퍼로 가치를 준다 → 자동 회신·일정 조율로 놓치지 않는다 → 사람은 면담과 제안에 집중한다. 이 일련의 흐름을 처음부터 하나의 구조로 설계한 것이 컸다.

## 구현 방법과 스킬

- 검색 상위를 딴 실제 LP를 15섹션짜리 완성 템플릿으로 범용화한 스킬 **pro-lp-builder**도 준비. 0에서 분위기만 나는 LP를 생성하는 게 아니라, SEO·구조화 데이터·반응형·보안까지 만들어 둔 완성 템플릿에 자사 정보를 갈아 넣는 방식.
- 블로그 자동화, AI 진단, 화이트페이퍼의 추가 구현 사고방식과 실제 프롬프트는 note에 정리(스킬 다운로드 안내 포함).
- 글 말미에는 LP 제작·운용 상담 폼과 법인 연수·컨설팅 문의 안내가 이어진다.
- ※ 수치는 2026년 7월 시점 사내 집계이며, 검색 순위는 시기·환경에 따라 변동하고 동일 성과를 보장하지 않는다.
